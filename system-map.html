Diagram 2: Authority & Pipeline Map

Three-layer architecture showing authority flow from governance (Layer-0) through internal execution (Layer-1) to static presentation (Layer-2). Includes gating concepts and approval boundaries.

Key Principles

  • Layer-0: Governance / Authority — Final decision-making power (trizel-core)
  • Layer-1: Internal Execution / Analysis — Private research and validation (trizel-lab + tools)
  • Layer-2: Presentation / Display — Static public-facing website (this site)
  • Gating — "Approved artifacts only" between Layer-0 and Layer-1
  • Boundary — "No execution / no analysis / no pipelines" enforced at Layer-2

External References Only — No Integration or Execution

All repositories shown in these diagrams are external references. This website (Layer-2) has no integration with any system, performs no execution, conducts no analysis, and displays no live data. All content is static HTML/CSS/SVG only.
