**角色：** 为整个 TRIZEL 系统建立政策、标准和治理框架。

**权限：** **最终且具有约束力**

## 不可协商的原则

TRIZEL 系统按照这些**不可变的原则**运作：

1. **严格的关注点分离**
   - 治理、执行和展示是独立的层级
   - 任何存储库都不履行两个角色
   - 边界是强制执行且不可协商的

2. **权限仅向下流动**
   - 层级-0 治理层级-1 和层级-2
   - 层级-1 通过验证的成果告知层级-2
   - 层级-2 展示但不解释

3. **公共存储库中无实时执行**
   - 层级-2（本存储库）仅包含静态成果
   - 无 JavaScript 执行、无 API 调用、无指标收集
   - 无构建过程或动态内容生成

4. **默认关闭发布**
   - 所有公共成果必须通过 Phase-E 网关
   - 暴露前需要人工批准
   - 默认状态为未发布

5. **需要人工批准**
   - 无自动发布研究成果
   - 公开前需手动审查和验证
   - 需授权人员的明确批准

## 这里可以发布什么

本存储库（`trizel-site-artifacts`）包含**仅限已批准的静态成果**：

### 允许的内容

- ✅ **静态文档**（Markdown 文件）
- ✅ **系统地图和图表**（SVG、PNG）
- ✅ **结构描述**（无解释）
- ✅ **权威来源引用**（链接到层级-0 和层级-1）
- ✅ **视觉识别资产**（标志、设计系统）

### 明确禁止

- ❌ **无代码执行**（无 JavaScript、无实时逻辑）
- ❌ **无分析或指标**（无跟踪、无计数器）
- ❌ **无动态内容**（无 API 调用、无实时数据）
- ❌ **无构建过程**（无打包器、无自动化）
- ❌ **无科学声明**（无研究结果）
- ❌ **无解释**（无层级-1 研究分析）

      ↓
Phase-E 网关

## 系统架构摘要

**权限结构：**
```
层级-0 (trizel-core)
   ↓ [治理]
层级-1 (trizel-lab、引擎、管道)

```

   ↓ [执行]
